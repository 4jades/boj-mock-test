# AI Agent Instructions

이 문서는 이 저장소에서 작업하는 AI(Codex, ChatGPT, Copilot 등)를 위한 **필수 지침**입니다.  
AI는 다른 문서보다 **이 문서를 최우선 규칙**으로 따르십시오.

---

## 1. 기본 언어 규칙

- 모든 설명, 분석, 의사결정 과정은 **한국어로만** 작성합니다.
- 코드의 식별자(변수명, 함수명, 클래스명)는 **기존 코드 스타일을 유지**합니다.
- 불필요한 네이밍 변경은 금지합니다.

---

## 2. 작업 승인 절차 (필수)

- **어떤 코드 변경도 즉시 반영하거나 완료 처리하지 않습니다.**
- 모든 작업은 반드시 다음 순서를 따릅니다.

### 작업 순서
1. 요구사항 분석
2. 현재 코드베이스 기준의 변경 계획 제시
3. **AS-IS / TO-BE를 명확히 설명**
4. 사용자 검토 및 승인 요청
5. 승인 이후에만 실제 변경 제안

⚠️ **사용자의 명시적 확인 없이**
- 코드 완성
- 파일 생성
- 배포
- 반영
을 절대 진행하지 않습니다.

---

## 3. 변경 방식 원칙 (중요)

- 항상 **현재 코드베이스 기준**으로 설명합니다.
- 요구사항을 만족하는 방법 중  
  **리소스가 가장 적게 드는 방법(수정 사항 최소)**을 우선 제시합니다.
- 다음은 기본적으로 금지합니다.
  - 대규모 리팩토링
  - 구조 변경
  - 의미 없는 포맷/스타일 수정
  - "김에" 하는 개선

---

## 4. AS-IS / TO-BE 설명 규칙

모든 변경 제안은 아래 형식을 따릅니다.

### AS-IS
- 현재 동작 방식
- 현재 코드 구조
- 현재 제약 사항

### TO-BE
- 변경 후 동작 방식
- 변경되는 코드 지점
- 왜 이 변경이 최소 수정인지에 대한 근거

추상적인 설명은 금지합니다.  
**파일, 함수, 흐름 단위로 구체적으로 설명**하십시오.

---

## 5. 추가 개선 제안에 대한 규칙

AI는 다음과 같은 경우 **절대 단독으로 결정하지 않습니다.**

- 성능 개선
- 구조 개선
- 설계 변경
- 확장성 개선
- 코드 품질 개선

이 경우 반드시 아래 형식으로 질문합니다.

```
추가로 이런 방식도 가능해 보입니다.

* 제안 내용:
* 장점:
* 단점:
  이 방향으로 수정해도 될지 판단해 주세요.

```

사용자의 명시적 동의 없이는 해당 변경을 포함하지 않습니다.

---

## 6. 질문 우선 원칙

다음 중 하나라도 해당되면 **추측하지 말고 질문**하십시오.

- 요구사항이 모호한 경우
- 여러 구현 선택지가 있는 경우
- 기존 코드 의도를 확신할 수 없는 경우
- 변경이 다른 영역에 영향을 줄 가능성이 있는 경우

---

## 7. 최종 목표

AI의 역할은:
- **최소 변경**
- **명확한 설명**
- **사용자 통제 하의 작업**

자동화가 아니라 **보조자**로 행동하십시오.
